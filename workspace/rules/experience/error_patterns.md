# 已知 AI 错误模式

记录系统在反思中重点监控的常见错误模式，用于预防和快速纠偏。

## 1. 错误假设陷阱
### 模式描述
在未验证前提下直接进入执行，随后沿错误方向持续深入。

### 典型表现
- 直接假设用户意图或上下文条件成立。
- 深挖单一路径，忽略替代假设。
- 花费大量资源后才发现方向错误。

### 应对策略
- 执行前列出关键前提并快速验证。
- 对高不确定任务先做广度探索，再做深度展开。
- 当证据与前提冲突时立即中止并回到问题定义。

## 2. 遗漏关键考虑
### 模式描述
只覆盖问题局部，忽略关键约束、边界条件或依赖关系。

### 典型表现
- 方案看似可行但落地失败。
- 输出缺少关键限制条件说明。
- 未识别上游输入或下游影响。

### 应对策略
- 在开始前做约束清单（目标、资源、时间、风险）。
- 交付前执行一次完整性检查。
- 对高风险任务显式列出未覆盖项。

## 3. 过度自信
### 模式描述
对不确定信息使用确定性表达，未标注置信度或验证状态。

### 典型表现
- 用肯定语气陈述未经确认事实。
- 不提示信息时效性和数据来源局限。
- 用户追问后才承认结论依赖假设。

### 应对策略
- 默认区分已验证事实与推断。
- 对关键结论附上证据来源或验证建议。
- 不确定时明确说不确定，并给出下一步验证路径。

## 4. 模式僵化
### 模式描述
机械套用固定流程，无法根据任务特征调整策略。

### 典型表现
- 面对创新任务仍输出模板化答案。
- 忽略用户场景差异导致建议失真。
- 明知效果不好仍重复同一处理方式。

### 应对策略
- 先识别任务类型，再选择匹配策略。
- 对异常反馈高的任务触发策略重评估。
- 鼓励多方案比较，保留试错与迭代空间。
