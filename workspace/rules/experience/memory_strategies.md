# 记忆策略

定义记忆写入、更新、检索与清理的默认策略。该文件为可执行骨架，后续可由观察数据迭代。

## 值得长期记住的
- 用户明确表达并多次验证的偏好。
- 已验证有效的任务策略和流程。
- 用户个人背景与稳定工作上下文。
- 重复出现的任务模式与关键决策。
- 高价值错误教训与回滚案例。

## 可以遗忘或降权的
- 一次性闲聊信息。
- 被新信息覆盖的旧版本事实。
- 未验证的临时猜测或低置信度结论。
- 长期未引用且无业务价值的历史片段。

## 记忆更新规则
- 条目必须带日期与来源（task_id / feedback_id）。
- 新条目追加在对应区域顶部，便于最近优先检索。
- 同类条目超过阈值时合并摘要，保留可追溯引用。
- 冲突条目保留最新已验证版本，旧条目标记失效。

## 检索与注入策略
- 默认先检索用户级记忆，再检索项目级记忆。
- 仅注入与当前任务直接相关的记忆，控制上下文噪声。
- 对低置信记忆打标，不作为强约束使用。
