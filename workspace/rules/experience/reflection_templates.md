# 反思模板

定义任务后反思的默认维度和输出格式。该文件为可执行骨架，运行后可持续补充。

## 轻量反思（每次任务后，Gemini 执行）
针对单次任务，用 1-2 句话回答：
1. 任务完成度：SUCCESS / PARTIAL / FAILURE
2. 分类：ERROR / PREFERENCE / NONE
3. 如果不是完全成功，核心原因是什么？
4. 本次是否产出可复用经验？
5. 用户反馈（若有）反映了什么偏好或问题？

输出格式：
{task_id} | {type} | {outcome} | {one_line_lesson}

## 深度反思（定时触发，Opus 执行）
按时间窗口回顾任务，重点分析：
1. 高成功率任务类型及其可复用做法。
2. 高频失败类型与根因归类（假设错误/遗漏考虑/工具误用/能力不足）。
3. 用户纠正是否呈现重复模式。
4. 记忆系统是否有效提供上下文。
5. 是否存在未触发或被忽略的规则。

输出建议：
- 关键发现（按优先级）
- 建议改动（1-3 条，含置信度）
- 是否进入信号系统（是/否，原因）
